Self-Hosted Cloud Server

Setting up a private, secure cloud storage solution using modern containerization and networking.

Project Goal

The primary objective was to deploy a private cloud environment to replace external services, focusing on data privacy, control, and resilience. This served as a practical exercise in system administration and networking within a Linux environment.

Technologies & Architecture

  • Linux (e.g., Ubuntu): Base OS for the server hardware.
  • Docker & Docker Compose: Used for containerization. This allowed easy deployment and management of Nextcloud, databases (like MariaDB), and reverse proxies.
  • Nextcloud: The main application providing file hosting, synchronization, and collaboration features.
  • Cloudflared Tunnel: Implemented a zero-trust network access model. This securely exposes the internal Nextcloud instance to the internet without requiring port forwarding, enhancing security significantly.

Key Learnings

  • Mastery of `docker-compose` for multi-container application orchestration.
  • Understanding of volume mapping for persistent data storage in containers.
  • Practical application of network tunneling for secure remote access (Zero Trust).
  • Basic system hardening and security best practices for public-facing services.
